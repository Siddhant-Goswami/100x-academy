<Problem>
You wrote code that reads a field off a response and it worked, until the day the
record was not there. Then the body came back empty and your code blew up reading
a key that did not exist. The reply was honest about it: the status said 404. Your
code just never looked. You need a check that decides, before touching the body,
whether there is a body worth touching.
</Problem>

<Concept>
The status code is a gate. A 200 means the body is yours to read. A 404 means
there is nothing behind it. Reading the body without checking the status is how
code breaks the first time something is missing. So the shape is: get the
response, compare the status to 200, and let that decide what you do next.
</Concept>

<Primitive name="The status gates the body">
Check the status before you read the body, every time. The status is the only
thing that tells you the body is real. Code that skips the gate is code that
trusts data that may not be there.
</Primitive>
