<Problem>
You have a service that returns a user as JSON. The page only needs the city.
Right now you are printing the whole object and reading it by eye. That does not
scale to a thousand users. You need code that asks for one user and hands back
one field, and that does something sensible when the user is missing.
</Problem>

<Concept>
A response is not just data. It carries a status that tells you whether the data
is even there. A 200 means the body is yours to read. A 404 means there is no
body worth reading. So the shape of correct code is: call, check the status,
then read the field.
</Concept>

<Primitive name="The request and response contract">
Every call returns a status and a body. The status decides whether the body is
real. Code that reads the body without checking the status is code that breaks
the first time something is missing.
</Primitive>
