<Problem>
The data your page needs lives on another machine. You cannot reach into that
machine and read its memory. All you can do is send it a message and wait for one
back. Before you trust anything in the reply, you need to know one thing: did the
other machine even agree to answer? That single fact decides whether the rest of
the reply is worth reading.
</Problem>

<Concept>
A request goes out, a response comes back. The first thing the response tells you
is a status code, a small number that says how the request went. 200 means the
machine answered and the data is there. Anything else means read no further until
you have handled it. So the very first move with any response is to look at its
status.
</Concept>

<Primitive name="Request out, response back">
You never read another machine's data directly. You send a request and read a
response. The response opens with a status code, and that code is what tells you
whether the rest of the response means anything.
</Primitive>
